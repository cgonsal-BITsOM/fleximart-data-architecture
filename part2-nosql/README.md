## Part 2 oveview
FlexiMart plans to expand its product catalog to support highly diverse product types (electronics, apparel, groceries, etc.) with varying attributes and embedded customer reviews.
The existing relational database model is optimized for uniform products.  MongoDB’s document data model aligns directly with FlexiMart’s exansion to diverse products
This repository evaluates MongoDB for this requirement and demonstrates core catalog operations.

nosql_analysis.md
Provides a structured analysis comparing relational databases and MongoDB for FlexiMart’s product catalog use case. It outlines RDBMS limitations, MongoDB benefits, and key trade‑offs in clear, business‑focused language.

mongodb_operations.js
Demonstrates core MongoDB operations against a sample products collection. Includes data loading, filtered queries, aggregations, updates to embedded reviews, and category‑level analytics.
